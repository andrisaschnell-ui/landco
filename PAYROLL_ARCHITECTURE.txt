Behavior Notes / Important Considerations
- Calculations:
  - Calculated fields are derived server-side in payroll_calculate_rows()
  - Summary rows (28–31) are totals/rollups and must remain read-only
  - Formula logic matches the Excel sheet; do not change without updating payroll_structure.docx
- Data safety:
  - “Delete Month Data” removes all payroll rows for that month/year
  - “Template Month” only clears admin-input fields (keeps calculated rows consistent)
- Import preview:
  - Preview shows differences in admin-input fields before applying
  - Apply import overwrites admin inputs for the target month/year
- Row controls:
  - "Clear Row" keeps NO but clears admin inputs
  - "Delete & Shift" removes a row and shifts admin inputs up (NO moves with employee)
- Display:
  - Data rows are sorted by NO column for display
  - Summary rows stay at the bottom
  - Summary rows are bold for visibility
- If you update formulas:
  - Update payroll_calculate_rows()
  - Update payroll_structure.docx (if used as spec)
  - Consider recalculating stored rows when formula changes

Future Plan: Salary Increase Settings (Yearly)
Goal: Allow admin to apply a percentage increase to salary base columns with an effective month.
Proposed approach:
1) Add a settings table, e.g. payroll_settings:
   - id, effective_year, effective_month, base_salary_increase_pct
2) Add UI inputs in Payroll page (or separate Settings page):
   - Percentage increase (e.g., 5%)
   - Effective month (1–12) and year
3) Apply logic in payroll_calculate_rows():
   - If month/year >= effective setting, compute:
     - Salario Base 2025 = Salario Base 2024 * (1 + pct)
   - Keep the original base salary column editable if needed
4) Update column headings each year:
   - Display dynamic label (e.g., “Salario Base 2026”)
   - Use settings to determine which year is active
5) Provide a “preview impact” on totals before saving.

Known Limitations / Future Work
- Auto-detect year from file name can fail for names without a year
- Import currently uses a minimal XLSX parser (no external libs)
- UI could add inline help for each column
- Consider computing totals live in UI for better UX
